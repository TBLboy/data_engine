# 数据标注工具调研

> 2026-07-14 | 调研目标：为平台后续扩展数据标注功能做技术选型
> 对标三个开源项目：LeRobot 官方标注工具、Label Studio + VLM Backend、RoboInter-Tools

---

## 一、背景

平台主流程（数据采集 → 存储 → 管理 → 质检 → 导出）中，**数据标注**是当前未覆盖但后续需要建设的环节。本次调研评估三个候选开源方案的可复用程度和迁移成本。

---

## 二、方案对比

### 2.1 LeRobot 官方标注工具

**定位**：HuggingFace LeRobot 生态内的标注能力，分两条线

**1. VLM 自动标注 Pipeline（`lerobot-annotate`）**

- 读取 LeRobot Episode 视频，调用 Qwen-VL 类 VLM 自动生成标注
- 标注内容：任务描述、连续子任务及时间边界、高层计划、任务重述、状态记忆、人机交互语句、视觉问答、bbox/keypoint/空间关系
- 两阶段流程：先描述视频 → 再划分子任务
- 结果直接写入 LeRobot v3.1 的 `language_persistent` 和 `language_events` 字段
- 许可证：Apache-2.0

**2. 人工标注 Web UI（`huggingface/lerobot-annotate`）**

- 加载本地或 HuggingFace Hub 上的 LeRobot 数据集
- 左侧选择 Episode，播放视频，添加/修改子任务起止时间
- 生成 `meta/subtasks.parquet`、`meta/tasks_high_level.parquet`
- 给数据帧写入 `subtask_index` 和 `task_index_high_level`
- **LICENSE 不明确**，商业使用需确认

**局限性**：
- 两套工具字段体系不完全一致（VLM 管道写 language_persistent/events，人工 UI 写 subtask_index），需额外转换层
- 前提是数据必须为 LeRobot 格式，公司数据需做全量格式转换
- 单用户使用，无多角色权限、无任务派发等生产级功能

### 2.2 Label Studio + VLM Backend

**定位**：通用开源标注平台，作为人工复核前端，自建 VLM 预标注后端

- 许可证：Apache-2.0
- 支持视频、图像、文本、音频、时间序列
- 标注界面可自由配置（TimelineLabels 可直接改造成机器人子任务标注界面）
- ML Backend 机制：模型包装为服务，结果作为预标注显示 → 人工检查/修改 → 提交
- 也支持离线 JSON 导入 predictions

**推荐工作流**：

```
LeRobot Dataset → VLM 预标注 → 导出 JSON predictions →
导入 Label Studio → 人工播放/拖动边界/修改 →
审核通过 → 导出 Label Studio JSON → 转换回 LeRobot Parquet
```

**标注员操作**：修改错误文字、拖动起止点、合并/拆分子任务、修改成功状态、选择失败原因、点击确认

**优势**：
- 成熟的多用户标注平台，内置项目管理、角色权限、任务分配、审核流程
- 架构与当前平台（FastAPI + PostgreSQL）兼容
- 不需要全量格式转换，只需写导入适配器

### 2.3 RoboInter-Tools

**仓库**：`github.com/InternRobotics/RoboInter`（ICLR 2026，MIT 许可证）

**架构**：Flask 标注服务端 + Qt 标注客户端（纯 Python）

**核心能力**：

1. **语言标注**：ChatGPT 预标注 → 人工标注（15 种 primitive skill 分类、任务分解、clip 分割、接触帧记录）→ 交叉校验 → 抽样验证
2. **分割标注**：SAM2 帧级 mask 生成，支持多轮迭代质控
3. **多用户支持**：标注池按 round-robin 分配

**目录结构**：

```
├── client/client.py          # Qt 标注 GUI（桌面应用）
├── server/server.py          # Flask 标注服务端
├── tools/
│   ├── parse_sam.py          # SAM2 批量处理 CLI
│   ├── sam_tools.py          # SAM2 核心函数
│   └── generate_annotation_pool.py  # 标注池生成
├── config/config.yaml
└── segment-anything-2/       # SAM2 模型
```

**可直接复用的模块**：
- `tools/parse_sam.py` + `tools/sam_tools.py`：SAM2 批量 mask 生成管线，可独立抽取
- 标注 schema 设计（`.npz` 格式、primitive skill 定义、contact frame 记录方式）

**无法直接使用的部分**：
- `client/client.py`（Qt 桌面 GUI，不能嵌入 Vue 3 前端）
- 标注数据模型需适配公司数据格式

## 三、推荐方案与后续路径

### 第一阶段：Label Studio 跑通基础标注

- 部署 Label Studio
- 写最小导入导出适配器（公司数据 → Label Studio task JSON → 回写）
- 先跑通子任务起止时间 / 成功/失败 / 失败原因等基础标注

### 第二阶段：接入 VLM 预标注

- 从 RoboInter 抽取 `parse_sam.py` + `sam_tools.py`，包装为 Label Studio ML Backend
- 参考 LeRobot `lerobot-annotate` 的 VLM 调用逻辑，生成预标注 JSON 导入 Label Studio
- 实现 Label Studio predictions 导入管道

### 第三阶段：高阶标注（按需）

- 参考 RoboInter 的 annotation schema 扩展高级标注类型（contact frame、primitive skill、affordance 等）
- 按需参考其 15 种 primitive skill 分类体系和接触帧记录方式

### 开源组件可复用性总结

| 组件 | 来源 | 复用方式 |
|------|------|---------|
| 标注平台骨架 | Label Studio | 直接部署（多用户、角色、任务分配原生支持） |
| SAM2 分割管线 | RoboInter `tools/` | 抽取为核心 Python 模块，包装为 ML Backend |
| VLM 调用逻辑 | LeRobot `lerobot-annotate` | 参考其 prompt 设计和两阶段流程 |
| 标注 schema 设计 | RoboInter + LeRobot v3.1 | 参考字段定义，适配公司数据模型 |
| 标注 UI | 自研（Vue 3）或 Label Studio 嵌入 | Label Studio 提供可配置前端，也可自行开发 |
| 3D URDF 可视化 | LeRobot Visualizer | 后续可视化需求可直接复用 |
| 格式转换 | 需自研 | 公司 TeleDex 格式 → 标注平台格式的适配器 |
