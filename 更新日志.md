# 更新日志

## 2026-07-07

### 新增
- **人工质检时间轴联动**：视频播放/拖动时，曲线图与异常段标签同步联动
  - 曲线图新增当前播放时刻竖线（蓝色虚线）和鼠标跟随竖线（黑色实线），两条游标均限定在绘图区域内
  - 点击曲线图任意位置可拖动 seek 视频，按住拖动时连续跳帧
  - 图例从 tooltip 改为顶部固定面板显示，鼠标不再弹出气泡
  - L3 异常时间段以背景高亮形式叠加在曲线图上（黄色=warn，红色=bad）
  - 异常段 chip 根据当前播放时刻同时高亮所有命中的标签（支持重叠异常段）
  - 当前命中的异常段在上方时间轴面板中以文本形式列出
  - 底部说明标签已精简，不再挤占曲线图空间
- **遥操作曲线双面板拆分**：删除异常段核查清单面板后利用垂直空间，机械臂/灵巧手模式各拆为两个上下排列的曲线面板（arm: 左臂7维 + 右臂7维，hand: 左手6维 + 右手6维）。图例统一在顶部仅显示左侧维度配色，左右面板共享配色。指针事件、游标、异常段覆盖层均适配双 chart 实例。
- **异常段背景色叠加逻辑**：改为逐像素列扫描——同一位置有多段重叠时颜色叠加加深；红色段覆盖黄色段（bad 优先）；红色底色更深 `rgba(220,38,38,...)` 增强对比度

### 修复
- **曲线图异常段背景首屏不显示**：`drawChartOverlay` 从 `chartInstance.draw()` 改为 `chartInstance.update('none')`，确保首屏加载时背景高亮直接渲染，无需缩放页面触发
- **异常段标签高亮只取第一条**：`activeTimelineSegment` 从 `find` 改为 `filter` + `Set`，重叠异常段全部同时高亮
- **曲线图游标线过长**：游标线高度改为绑定 `chartArea` 的 `top` 和 `bottom`，不再穿过图例区域
- **DOF 维度检测修复**：后端 `_detect_dims()` 启发式扫描改为读取 metadata.json → hardware_specs 中的 `left_dof`/`right_dof`，修复 handDims 只返回 6 而非 12 导致灵巧手每侧只显示 3 DOF 的 bug

### 变更
- **取消游标联动高亮**：异常段背景色不再随播放游标位置动态变化，改为固定底色
- **删除异常段核查清单面板**：移除 el-card + el-check-tag 组件，清理相关死代码
- **维度标签修正**：机械臂按钮显示 `armDims/2`（14→7 维）

## 2026-07-06

### 修复
- **扫描入库填充时长/帧数**：MinIO 扫描时自动读取 manifest.json 提取 duration 和 frame_count，写入 Episode 和 EpisodeInventory，修复数据总库页面所有 episode 时长/帧数始终为 0 的问题。
- **定时扫描时区修正**：APScheduler CronTrigger 增加 timezone='Asia/Shanghai'，每日自动扫描从 UTC 0:00 改为北京时间 0:00 触发。
- **审核锁重复显示**：任务池页面审核锁列移除冗余纯文字，仅保留 el-tag。人工质检页面无人认领时不再重复显示。
- **人工质检锁面板样式修复**：scoped CSS 覆盖全局 flex 布局导致内容垂直堆叠、背景色不匹配。恢复 flex 居中布局及透明背景，新增 lock-actions 按钮对齐。
